# 聊天截图与日志分析：为何回复「没看到订单图」

## 截图与日志对应关系

| 截图 | 日志 |
|------|------|
| **AI 智控土** 发了一张 easypaisa 付款凭证（Rs. 300, Transaction ID#41504924558） | 日志里**没有**「收到消息 [甲方模拟回复群/xxx]: [图片]」—— 说明**图片那条消息**要么没触发回复、要么是别的账号发的，本端只处理了**文字消息** |
| **A智** 发「看到订单图了吗」 | `04:11:44` 收到消息 [甲方模拟回复群/ai_zkw]: **看到订单图了吗** |
| **AI 智控王** 回复「哎呀，我这边还是没看到订单截图呢…麻烦再发一下」 | `04:11:47` 已回复消息: **哎呀，我这边还是没看到订单截图呢…** |

结论：截图里的回复就是当前程序在 04:11 那次的真实回复，和日志一致。

---

## 原因分析

### 1. 图片和问句不在同一条消息里

- **订单图**是 **AI 智控土** 在**上一条（或更早）**消息里发的。
- **「看到订单图了吗」**是 **A智** 在**当前**消息里发的纯文字。
- 程序只会对「当前这条消息」做处理：当前消息只有文字、没有附件，所以**不会**对任何图片做 OCR。

### 2. 当前逻辑只对「本条消息的图」做 OCR

- 带图+说明：只对本条消息里的图做 OCR，结果放进 `image_ocr_text`。
- 纯图：本条消息当「图片识别」处理。
- **没有**「看看群里最近有没有人发图，拿来 OCR」的逻辑。  
所以当用户问「看到订单图了吗」时，程序拿到的只有这句文字，没有任何 OCR 结果，于是 AI 按 system prompt 如实说「没看到订单截图，请再发一下」。

### 3. 为何日志里看不到「图片消息」

- 若图片是 **AI 智控土** 发的，且群组是「关键词/@ 才回复」，那条纯图消息可能**没触发**回复，因此不会进入「收到消息 → 触发 → 处理」的流程，日志里就只有后续 **ai_zkw** 的「看到订单图了吗」。
- 即使触发了，我们也没有把「别人刚发的图」和「当前用户问看到图了吗」关联起来，所以从回复逻辑上仍然等同于「没图可看」。

---

## 解决方案（已实现）

在**群聊**里，当**当前消息没有图**且用户消息包含「订单」+「看/图/发」时：

1. **拉取该群最近 15 条消息**（`get_chat_history(chat_id, limit=15)`）。
2. **找到其中最近一条带图的消息**（不限发送者），下载并做 OCR（`_get_recent_chat_image_ocr`）。
3. 将 OCR 结果写入 **`image_ocr_text`**，与当前文字一起传给技能与 AI。
4. AI 收到「用户刚发的图片/截图内容（OCR）」后，可回复「看到了，是这张吧？…」并依据 OCR 概括，不再说「没看到」。

**代码位置**：`src/client/telegram_client.py`  
- `_get_recent_chat_image_ocr(chat_id, limit=15)`：拉取群内最近一条带图消息并 OCR。  
- `_process_message_async`：当 `image_ocr_text` 为空且文本为订单相关问句时，调用上述方法并补全 `image_ocr_text`。

这样既不会编造订单（仍以 OCR + 机器人通知为准），又能在「图在上一条、问在本条」的场景下正确「看到」订单图。
