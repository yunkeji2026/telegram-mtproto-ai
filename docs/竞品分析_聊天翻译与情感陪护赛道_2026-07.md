## 0. 一句话结论

**六家竞品没有一家和我们打同一场仗**：云译/海王/拓译/OneChat 卖「**人在聊，工具帮翻译**」的效率工具（按字符/端口收费，$4.2–9/端口/月红海）；EngageLab 是官方 API 合规企业触达（另一战场）；DeepL 是我们 `EngineRouter` 的上游线路而非对手。我们的身位是「**AI 以人设替人聊，人只审核接管**」+ 情感陪护纵深——代差级差异。

我们真正的输点不在能力，在**交付形态**：竞品「下载 exe → 扫码 → 10 分钟开聊 → 免费额度 → 充值」的自动化漏斗，对我们「TG 客服人肉 + USDT + 手工部署」是数量级碾压。**P0 全部围绕可售性，不新造能力。**

最值得盯的对手是 **OneChat**：已把「人设回复/对话记忆/语音克隆/换脸/养号」写上官网（个人版永久免费抢量），只是深度停在「建议式/单点功能」。它走到「全自动替聊」估计 6–12 个月——这是我们的窗口期。

## 1. 六竞品画像与定价

### 1.1 云译 YunYi（工具型 · 翻译外挂旗舰）

出海社交聚合翻译器，Windows 客户端。六引擎（有道/百度/谷歌/DeepL/ChatGPT/Gemini）实时双向翻译、音视频/图片翻译、AI 智能回复、打粉计数器（工单）、群发、代理 IP、禁发中文拦截、子账号。计费：按字符阶梯（基础/高级/专业）+ 包月双轨。
→ 深扫见 [`竞品分析_云译YunYi_与实时翻译赛道_2026.md`](竞品分析_云译YunYi_与实时翻译赛道_2026.md)；其翻译 UX 差距已被 P55–P61 大幅追平。

### 1.2 DeepL（上游引擎 · 质量基准，非对手）

企业级翻译平台：文本/文件/语音/API + 术语表 + Write 润色，20 万+ 企业客户。
→ **定位关系**：是我们 `EngineRouter` 的一条线路（`src/ai/translation_engines.py::DeepLEngine`），同时是用户对「翻译质量」的心理基准。我们的「AI 拟人翻译」叙事必须能当场演示出比 DeepL 直译更「像本地人」，否则卖点不成立。对外话术：「DeepL 是我们的线路之一，我们在其上做拟人化与术语强制」。

### 1.3 海王出海（工具型 · 规模化价格战）

SCRM 聚合平台，10+ 平台多开 + 指纹环境 + 会话级 IP 代理。翻译线路最全：谷歌/DeepL/微软/必应/有道/火山/Yandex + GPT/Deepseek/Gemini（不限制）。工单重粉标记、粉丝查重、进粉实时记录、团队权限、分流链接统计。
**定价锚点极低**：字符版 $24/150 万字符不限时（$1068/亿字符）；包月 $40/月/5 端口起，梯度到 **1000 端口 $4200/月（$4.2/口）**。
→ 启示：工具层单价已被打穿，**不要进这一层比价**；但「指纹/IP/查重/团队权限」是这个客群的心智标配，我们对应能力（proxy/fingerprint 注入、防关联）要显性化。

### 1.4 拓译 Tranlico（工具型 · 功能面重叠最多）

「端口永久免费、应用无限多开」引流。18+ 平台，200+ 语种，10+ 翻译线路**可对照选译**；AI 角色推荐回复（可导入历史训练）；**文字转语音条发送**（多音色多语速）；智能工单全流程；客户信息继承（离职/封号客户不丢失）。
→ 启示：①「多线路对照」被它教育成坐席习惯（我方 F+ 会话引擎偏好已基本追平，剩可视化增强，见 ROADMAP #14）；② **TTS 语音条是被验证的刚需**——我们有克隆音 TTS 全链路，体验上限更高，但要接到收件箱一键可发（P0-6）；③「客户资产不丢」话术直击老板痛点，我们 contacts/journey/360 更强但没这样卖（P0-5）。

### 1.5 EngageLab / 极光（企业触达 · 不同战场）

全渠道客户互动平台：AppPush/WebPush/SMS/Email/WhatsApp Business API/OTP/营销自动化/LiveDesk。企业订阅、官方 API、合规叙事（GDPR）、高送达率高并发。
→ 定位关系：它做「企业对客户的合规批量触达」，我们做「个人号对客户的拟人深聊」。短期不进它的战场（企业认证/模板审核/招投标）；长期客户长大后怕封号时，「官方 API 白牌合规版」是留住升级预算的出口（我方官方三端 + IG/Zalo webhook 已有基座）。ROADMAP #15「营销 journey 编排」对标于此。

### 1.6 OneChat 一聊（最接近的对手 · 重点盯防）

36+ 平台聚合、AI 翻译（宣称 98% 准度、100+ 语言）、**AI 对话记忆 + 人设回复**、24h AI 助手、**AI 语音克隆**、**AI 换脸/一键换装**、**智能养号**（模拟真人行为降封号）、100% 本地存储隐私叙事。**个人版永久免费，团队版按席位。**
→ 它已经把我们的差异化词汇全写上官网，但深度停留在「点一下生成建议」级：无全自动闭环、无受控出站/人审、无情绪-危机安全链、无亲密度/主动触达、无评测门禁。**防守要点**：用「全自动 + 陪护纵深 + 可验证安全指标」拉开身位，并警惕它的免费漏斗吃掉工具层增量用户。

## 3. 三视角分析

### 3.1 开发工程师视角

**不用动的强项**：门禁化工程体系（哑按钮/孤儿引用/i18n ratchet/路由契约/517 个测试文件）、危机安全链路、翻译术语跨引擎跨模态强制、能力分阶段开闸治理、观测栈（metrics→ops 看板→Prometheus）。这些是竞品十人团队一年也补不出来的底子。

**真实差距清单（按修复价值排）**：

1. **交付形态是第一瓶颈**。竞品「下载 exe→扫码→开聊」10 分钟；我们要 Python 环境 + config + RPA/真机。`desktop/` 已有 Electron 壳 + sidecar 骨架，缺「单文件安装包 + 首启向导」最后一公里。**能力再强，装不上等于零。**（→ P0-1）
2. **商业化通道为零**。License/席位有基座（M 线），但无试用签发、无字符额度、无自助开通——竞品的「注册送 2 万字符」获客漏斗我们完全没有。（→ P0-4）
3. **协议链路未真号验证**是最大技术杠杆。TG MTProto / WA Baileys 骨架已在：验证通过则单机跑几十协议账号，成本与稳定性碾压「客户端嵌网页版多开」的所有竞品；停在 RPA 真机则规模化成本反而更高。（→ P1）
4. **翻译 UX 已基本追平并局部反超云译/拓译**（P55–P61 + Phase F）：出站多引擎对照弹窗、会话引擎偏好落库+徽标已交付（ROADMAP #10/#14 为文档漂移，代码已在）。剩余增量：置信度评分暴露到 UI、入站消息对照、手动引擎下拉、入站自动译默认策略双链路收敛。（→ P0-2/P0-3）
5. 已知坑：向量记忆「假开」（开 flag 未配 embedding 静默退化关键词）、单进程假设卡多租户、四平台选择器校准依赖真号 [需外部]。

### 3.2 市场经理视角

**市场分三层**：

- **工具层**（人聊，工具译）：云译/海王/拓译。海王打到 $4.2/端口/月——**没有利润，不正面进**。
- **AI 辅助层**（AI 给建议，人点发送）：OneChat 领跑（免费抢量），工具四家跟进 AI 建议。靠席位，利润一般。
- **AI 替聊层**（AI 以人设全自动接客/陪护，人管异常）：**只有我们**。一个 AI 坐席替代 0.5–1 个人力（月成本 $500–1500），198–598 USDT/月 的定价空间真实存在——前提是以「AI 员工」而非「翻译工具」计价与叙事。

**定位语**：不说「聚合聊天翻译软件」（会被拉进 $7/端口比价），说「**多语种 AI 员工：以你的人设 7×24 接客、维系、成交，人工只管接管**」。翻译是器官，不是名字。

**竞品启示按「抄 / 防 / 避」**：
- **抄**：客户端即用、免费字符额度、「客户资产不丢」话术、「智能养号」话术（我们有实底：canary/节奏闸门/指纹）、下载页三步上手文案。
- **防**：OneChat 往人设/语音克隆/养号方向走——用「全自动 + 陪护纵深 + 可公开验证的安全指标」拉开身位；把评测门禁数据（人设一致性召回 1.0、危机 severe 召回 1.0、回译质量分、语音语种一致性）产品化为对外指标页——这是所有竞品给不出的信任资产。（→ P1-4）
- **避**：EngageLab 的合规企业市场短期不进；DeepL 不对标只借力；海王的千端口价格战不接。

**获客漏斗改造优先级**＝P0-1（装得上）→ P0-4（免费试到）→ P0-5（卖点看得见）→ 官网下载页（website/ 已有落地页体系，加 ChatX 下载/试用页）。

**合规与措辞红线**：赛道词汇体系贴着灰产（「打粉」「防封」「群发」）。营销锚定「跨境私域运营 / 情感陪伴产品运营商」；把危机安全网、审计日志、C2PA 做成信任卖点；「无审查 AI」「看不出破绽」类文案在获客页降权（渠道封禁风险 + 吓走正经客户）。仓库既有北极星「不烧号陪伴」（ROADMAP「刻意不做」段）与此一致：**不做大规模群发/批量加粉**，批量触达只走已交付的合规再激活闭环（四重闸 + 回复率 ROI）。

### 3.3 用户视角

**A. 工作室老板（买单人）**：
- 「员工走了/号封了，客户还在不在？」→ 数据本地 store + contacts 跨平台身份 + 360 视图全都有，**要包装成『客户资产保险箱』并补『换号续聊』操作路径**（P0-5）。
- 「一个人能管几个号、成单率多少？」→ 今日计数条/agent_perf/ROI 页已有，缺「老板日报」推送形态（P1 备选）。
- 「封号率多少？」→ 我们有真实的节奏闸门/canary/看门狗，**要给出『养号模式』开关与数据**；OneChat 只有话术，我们能有数字（P0-5）。

**B. 一线坐席**：
- 「翻译哪条准？」→ 多线路对照习惯已被竞品教育；我方出站对照弹窗 + 引擎偏好徽标已有，缺置信度评分可视化与入站对照（P0-2）。
- 「懒得打字」→ 拓译验证了 TTS 语音条刚需；我们克隆音上限更高且协议账号已能发，缺 RPA 三端桥接与一键化 UX（P0-6）。
- 「忙不过来」→ autosend + 受控出站是解，需要「AI 值守」按钮化（值守中/仅建议/关闭），从配置文件操作变成产品操作（P1）。

**C. 终端聊天对象（陪护模式的「客户」）**——体验 = 不穿帮，拆解为六件事，全部有实现、多数默认关：
翻译像本地人（拟人翻译）｜人设不自曝（persona_guard + 门禁）｜记得住上次的话（记忆链路，向量召回待配 embedding）｜语音是同一个人（克隆 + 语种一致门禁）｜作息像真人（主动触达 + 情绪闸门）｜要照片给「本人」照片（人设相册）。
**这条体验链竞品全线为零，但只有真号长期运行才能验证**——P1 真号灰度是唯一路径。

## 4. 优化路线

### P0 — 可售性冲刺（0–8 周，全部 [自治] 可并行）

> 原则：**不新造能力，把已有能力变成「装得上、试得到、看得见」**。
> 工程拆解（任务卡级，含文件锚点/验收/规模）见 [`P0_可售性冲刺_工程任务拆解_2026-07.md`](P0_可售性冲刺_工程任务拆解_2026-07.md)。

| # | 事项 | 一句话目标 | 现状底座 |
|---|---|---|---|
| P0-1 | 桌面安装包 + 首启向导 | 安装→AI Key→接第一个号→翻译生效 ≤10 分钟 | electron-builder NSIS + PyInstaller sidecar 已在，缺向导串联（底座 ~60–70%） |
| P0-2 | 翻译对照体验增量 | 置信度徽标进对照/单译结果 + 入站消息对照 + 手动引擎下拉 | 对照弹窗/引擎偏好落库+徽标已交付（Phase F/F+/F+2） |
| P0-3 | 入站自动译默认策略 | 有引擎即默认开（按会话可关），双链路收敛 + 成本护栏 | 前端懒翻默认 zh 已开，后端 enrich 默认关，双轨待收敛 |
| P0-4 | 免费试用通道 | 试用 license 带字符额度 + 扣减强制 + 半自动发卡 + 粘贴激活 | Ed25519 license/席位/只读锁已闭环（~70%），缺字符计量与发卡 |
| P0-5 | 「客户资产保险箱」+「养号模式」显性化 | 已有 contacts/风控能力 → 预设档 + 看板卡 + 营销页 | contact360/合并/CSV、预热爬坡/canary/Kill-Switch 全在，缺 UI 与包装 |
| P0-6 | 收件箱一键发克隆音语音条 | 合成+发送一键化；RPA 三端桥接 | 协议账号 tts-test→send-voice 闭环已在，RPA 501 待桥 |

### P1 — 拉开代差（8–16 周）

1. **协议账号真号灰度**：TG MTProto / WA Baileys 各 3–5 号跑 30 天，产出封号率/在线率数据（ROADMAP Tier1 D4 真号侧 + 四平台选择器校准同窗推进）。[需外部]
2. **「AI 值守」一键化**：L2 autosend 双 opt-in + 受控出站包装为收件箱开关（值守中/仅建议/关闭），复用 companion capability 预设档。[自治]
3. **陪护纵深真号验证**：向量记忆配真实 embedding（消「假开」坑）、主动触达灰度、人设相册实测轮播。[需外部]
4. **可信指标对外页**：`scripts/run_eval --json` 自动出报告页（人设一致性/危机召回/回译质量/语音语种一致性）——竞品给不出的信任资产。[自治]
5. **平台覆盖口径**：IG/Zalo 官方通道转正宣传 + Discord 评估（认知差 6 vs 36 要缩小）。
6. **拟人翻译对照素材流水线**：同句「DeepL 直译 vs 拟人译」批量产出，喂营销站。[自治]

### P2 — 护城河（16 周+）

- **真多租户 SaaS**（ROADMAP Tier3 #9，大版本）：解锁 OneChat 式「个人免费→团队付费」漏斗；此前维持私有化高客单。
- **实时语音通话陪护** + **数字人视频消息**（ROADMAP #11，[需外部] GPU/语音主机）：兑现营销站承诺或降口径。
- **官方 API 白牌合规版** + journey 编排（ROADMAP #15）：接住长大后怕封号的客户预算。

### 明确不做

- 不进 $4–9/端口纯工具价格战；不堆 10 引擎（3 引擎+置信切换+术语强制+拟人化已是差异化，堆量是伪需求）。
- 不做大规模群发/批量加粉/「打粉」词汇体系的功能（与「不烧号陪伴」北极星冲突；等价数据用今日计数条 + 合规触达 ROI 提供）。

## 5. 风险提示

1. **窗口期**：OneChat 从「建议式人设」到「全自动替聊」约 6–12 个月；P0 必须在此之前完成，否则代差被「够用 + 免费」抹平。
2. **RPA/个人号 TOS 风险**是结构性的（全行业共担）：Kill-Switch/canary 是缓解不是消除；协议化 + 官方 API 双轨是长期解。
3. **陪护业务伦理暴露面**大于翻译工具：危机安全网既是道德底线也是商业护城河，任何砍安全链路换性能的提议应拒绝。
4. **营销承诺 vs 交付能力缺口**：营销站五条产品线（换脸/数字人）与本仓库实际集成深度有差，要么兑现（P2）要么降口径，否则售后信任反噬 ChatX 主线。
