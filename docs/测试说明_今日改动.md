# 今日改动测试说明

本文说明如何验证「关闭私聊回复」「去除 claude-4.6-oups-high 人设与表述」以及「对话命令改额度配置」的测试方法。

---

## 一、今日改动摘要

| 改动 | 说明 |
|------|------|
| **私聊不再回复** | `process_private: false`，且代码仅在为 true 时注册私聊处理器；该账号只回复群组消息。 |
| **去除 claude-4.6-oups-high 表述** | 配置注释、日志、文档中「claude-4.6-oups-high」改为「AI / 大模型」等中性表述；系统身份仍为 Camille 客服。 |
| **对话命令（已有功能）** | 与机器人发「列出特殊群」「添加特殊群 群名」等可改 `quota_rules.yaml`，需先开启并配置白名单。 |

---

## 二、测试前准备

1. **启动程序**（在项目根目录）：
   ```bash
   cd telegram-mtproto-ai
   python main.py
   ```
2. **看启动日志**，确认：
   - 出现 `私聊消息处理已关闭 (process_private=false)，仅处理群组消息` 或 `消息处理器已设置 - 私聊=关, 群组=开` → 私聊关闭生效。
   - 出现 `✅ AI 客户端初始化成功`（不再出现「claude-4.6-oups-high」）→ 表述已替换。
   - 无报错、Telegram 登录成功。

---

## 三、测试 1：私聊不回复

1. 用**另一个 Telegram 账号**（或同事账号）给本账号**发私聊**，例如：
   - 「你好」「在吗」「测试」
   - 「订单查一下」「当前额度多少」
2. **预期**：本账号**不回复**任何私聊。
3. **可选**：把 `config/config.yaml` 里 `process_private` 改为 `true`，重启程序，再发一条私聊，应能收到回复；改回 `false` 并重启后，私聊 again 不回复。

---

## 四、测试 2：群组照常回复

1. 把本账号拉进一个**测试群**（或使用已有群）。
2. 在群里用以下方式之一触发回复（取决于 `group_reply.mode`，当前多为 `mention_or_keyword`）：
   - **@ 本账号** 并发一句，如「在吗」或「查下订单」；
   - 或发**含关键词**的消息，如「客服」「订单」「额度」「camille」等（关键词见 `config.yaml` 里 `group_reply.keywords`）。
3. **预期**：本账号在群里**有回复**，且身份/话术是 Camille 客服（订单、额度等），**不会**出现「claude-4.6-oups-high」「开发者」「通用聊天引擎」等表述。
4. 若未回复：看日志里是否有 `[群组监控] 跳过未触发消息`，确认是否未命中 @ 或关键词；可开 `logging.level: DEBUG` 再试。

---

## 五、测试 3：对话命令改额度配置（可选）

仅在需要测试「通过聊天改特殊群/黑名单」时做：

1. **开启并配置**：
   - 在 `config/config.yaml` 中设 `telegram.quota_config_commands.enabled: true`；
   - 在 `allowed_user_ids` 中填入你的 Telegram 用户 ID（可先发一条消息给 @userinfobot 获取）。
2. **重启** `python main.py`。
3. 用该账号在**私聊或群聊**里给本账号发（当前私聊已关，需在**群聊**里 @ 本账号发）：
   - `列出特殊群` → 应回复当前特殊群列表；
   - `列出黑名单` → 应回复当前黑名单或空；
   - `添加特殊群 测试群A` → 应提示已添加，且 `config/quota_rules.yaml` 里 `special_groups` 多出「测试群A」；
   - `删除特殊群 测试群A` → 应提示已删除。
4. **预期**：回复符合操作结果，且 `quota_rules.yaml` 被正确写回；再次「列出特殊群」能看到变更。
5. 用**不在** `allowed_user_ids` 的账号发同样命令 → 应无配置修改或提示无权限。

测完后若不需要对话改配置，可将 `enabled` 改回 `false` 并重启。

## 六、测试 4：日志与界面无 claude-4.6-oups-high

1. **日志**：查看 `logs/app.log`（或控制台），搜索「claude-4.6-oups-high」→ **不应再出现**；应看到「AI 客户端」「AI API」「AI 熔断」等。
2. **对话内容**：在群内多问几句（订单、额度、你是谁等），确认回复里**没有**「claude-4.6-oups-high」「claude-4.6-oups-high 团队」「通用聊天引擎」等，只有 Camille 客服话术。
