# AI 多语种聊天陪护平台：试点交付清单

更新日期：2026-05-31

本清单用于本地/私有化试点交付，目标是在客户环境中用 3-5 个账号跑通真实聊天。

## 1. 试点目标

- 一个运营人员可在统一工作台中管理 20-50 个活跃对话。
- 四个平台进入同一页面：Messenger、WhatsApp、Telegram、LINE。
- 每条消息可查看语言、翻译、AI 分析、回复建议和对象档案。
- 回复前可选择人工发送、人工审核、多答案选择、AI 自动模式。
- 文本回复可生成语音预览。
- 高风险内容不进入默认自动发送。

## 2. 环境准备

- Windows 10/11 或可运行现有项目的服务器。
- Python 环境与项目依赖已安装。
- 已配置 `config/config.yaml`。
- Web 后台可访问。
- 至少准备 1 个测试运营账号。
- 至少准备 3 个真实或测试聊天账号。

## 3. 平台接入检查

| 平台 | 第一版要求 | 验收方式 |
| --- | --- | --- |
| Messenger | 可读取待回复消息，可发送文本 | 工作台可看到并回复 |
| WhatsApp | 可读取 pending 消息，可发送文本或进入原有审批流 | 工作台可看到并尝试回复 |
| Telegram | 可读取最近消息，可发送文本 | 工作台可看到并回复 |
| LINE | 可读取聊天列表，可发送文本或进入原有发送队列 | 工作台可看到并尝试回复 |

## 4. 演示流程

1. 打开 Web 后台。
2. 进入统一收件箱。
3. 左侧选择一个非中文聊天对象。
4. 查看中间聊天记录。
5. 点击“翻译为中文”。
6. 查看右侧 AI 助手：
   - 情绪
   - 意图
   - 风险等级
   - 对象档案
   - 多答案建议
7. 点击一条建议“填入输入框”。
8. 点击“发送前翻译预览”。
9. 点击“生成语音预览”试听。
10. 切换回复模式：人工发送、人工审核、多答案、AI 自动。
11. 发送一条测试文本。

## 5. 试点验收指标

| 指标 | 目标 |
| --- | --- |
| 统一工作台可见平台 | 4 个 |
| 单运营可维护对话数 | 20-50 个 |
| AI 回复建议可采纳率 | 60% 以上 |
| 新消息到分析结果展示 | 10 秒内 |
| 回复从建议到发送 | 3 次点击以内 |
| 语音预览生成 | 30 秒内，失败可回退文字 |
| 高风险自动发送 | 0 |
| 连续运行 | 7 天 |

## 6. 风险边界

- 不承诺真人恋爱关系。
- 不鼓励欺骗身份。
- 不做无授权声音克隆。
- 金钱、隐私、成人、危险、自伤等内容必须人工审核。
- 用户明确要求停止联系时，必须停止主动触达。
