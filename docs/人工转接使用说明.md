# 人工转接 — 功能与使用说明

> 用户在群里反复问同一个问题时，机器人会自动 @ 人工客服来处理。
> 下面用最简单的话说清楚每个功能怎么用。

---

## 一、这个功能做什么

当一个用户在群里**短时间内反复问同一个问题**（比如 10 分钟内问了 3 遍），机器人在正常回复之后会**自动追加一段文字**，@ 人工客服来接手。

同时：
- 把用户那条原话**转发到客服的私聊**里。
- 再发一条**带按钮的消息**，客服点一下就能跳到群里那条消息的位置。

---

## 二、最快上手（三步）

打开管理后台 → 左侧「人工客服转接」 → 展开。

| 步骤 | 做什么 | 填什么 |
|------|--------|--------|
| ① | 勾选「启用」 | 打勾即可 |
| ② | 填客服用户名 | 填你的客服 Telegram 用户名，**不要带 @**，比如 `support_bot` |
| ③ | 点「保存转接配置」 | 保存后立刻生效，不用重启 |

> 点「一键常用默认值」可以自动帮你填好：启用 + 3 次触发 + 10 分钟窗口 + 5 分钟冷却 + 始终可 @。你只需要填客服用户名。

---

## 三、所有功能一览

### 1. 触发条件

| 设置项 | 意思 | 默认值 |
|--------|------|--------|
| 同一问题连发几次触发 | 用户问同一个问题问几遍后 @ 人工 | 3 次 |
| 统计窗口（分钟） | 在多长时间内计算重复次数 | 10 分钟 |
| 最短问句长度 | 太短的话（比如「嗯」）不算 | 6 个字 |
| 触发后冷却（分钟） | 已经 @ 过人工后，多久内不会再 @ | 5 分钟 |

**说明**：「同一个问题」是指意思一样的话，句末的标点符号不同（比如「订单到了吗」和「订单到了吗？」）会被当成同一个问题。

### 2. 客服设置

**简单模式（一个客服）**：
- 在「客服用户名」里填一个名字就行。
- 如果你知道客服的 user_id（一串数字），填上更好（@ 的链接更稳定）。不填也能用。

**多个客服**：
- 切换到「完整」视图，在客服表格里一行一行添加。
- 每个客服至少要有**用户名**或 **user_id** 其中一个。

### 3. 值班模式（什么时候可以 @ 客服）

| 模式 | 意思 |
|------|------|
| 始终可 @（默认） | 任何时候都会 @ 客服 |
| 仅手动值班 | 只有在后台手动勾选「值班中」时才 @ |
| 仅工作时间段 | 按你设定的上下班时间来（比如周一到周五 9 点到 18 点） |
| 时间段 或 手动 | 上班时间内，或者手动开了值班，都会 @ |
| 时间段 且 手动 | 必须**同时**在上班时间内**并且**手动开了值班，才会 @ |

> 大多数人选「始终可 @」就够了。要按上下班时间来，展开「值班 / 排班」填写即可。

### 4. 非值班时怎么办

如果触发了转接但当前不在值班时间：
- **填了「非值班提示」**：会回复这段话，比如「当前非工作时间，请稍后再试」。
- **没填**：什么都不追加，当作没触发。

### 5. 触发后群里会显示什么

机器人正常回复用户之后，会在最后追加一段话，大概长这样：

```
────────
已多次询问同类问题，请人工客服协助：@客服用户名

原文：用户的原话（可点击跳到该条消息）
```

- 「@ 前说明文案」可以改成你想要的话。
- 用户的原话会做成**可点击的链接**，客服点了就能跳到群里那条消息。

### 6. 私聊转发给客服

触发 @ 人工后，机器人还会做两件事：

**第一件：转发原话**
- 把用户在群里说的那条消息**原样转发**到客服的私聊里。
- 客服能看到「转发自某某群」。

**第二件：发一条定位消息**
- 在转发之后再发一条消息，里面有：
  - 群名
  - 一条**可以点击的链接**，点了直接跳到群里那条消息
  - 一个**按钮**「📍 打开群内该条消息」，点了也能跳过去

> 为什么要发两条？因为有些手机上，点转发条不一定能跳回群里。多一条带链接和按钮的消息，**怎么点都能跳过去**。

**怎么关**：
- 不想转发原话：取消勾选「触发转接后，把用户该条群消息转发到客服私聊」。
- 不想多发那条定位消息：取消勾选「转发后再发一条私聊：内联按钮 + 直达链接」。

### 7. 冷却机制

触发一次 @ 人工之后，同一个用户**问同一个问题**，在冷却时间内不会再次 @。

- 默认只对**同一句话**冷却。用户换一句新问题反复问，达到次数后**还是会再 @ 人工**。
- 如果你想：不管问什么，@ 过一次后统一冷却 → 需要改配置文件把 `escalation_cooldown_scope` 设成 `per_user_chat`。

## 四、进阶功能（一般不用动）

### 分组排班

如果你有**多组客服**（比如白班和夜班），可以在「完整」视图里设置：
- 每组有自己的客服列表和上班时间。
- 当前时间在谁的班，就 @ 谁的组。
- 没有任何组在班时，默认回退到总客服列表。

### @ 方式

| 选项 | 效果 |
|------|------|
| 全部 @（默认） | 每次 @ 所有客服 |
| 每次只 @ 一名（轮询） | 按顺序一个一个来 |
| 每次只 @ 一名（随机） | 随机选一个 |

### 工作时间设置

- **周模板**：比如设周一到周五 9:00–18:00。
- **例外日**：比如设 2026-01-01（元旦）全天休息。例外日会覆盖周模板。
- 支持跨午夜的时段，比如 22:00–06:00。

### 手动值班开关

在设置页最底部有一个「手动值班」勾选框：
- 勾上 = 现在值班中。
- 取消 = 现在不值班。
- 只在值班模式包含「手动」时才有用。

## 五、常见问题

### 用户反复问了但没有 @ 人工？

按这个顺序排查：

1. **有没有启用**？确认勾选了「启用」并且保存了。
2. **客服有没有填**？至少要填一个用户名或 user_id。
3. **机器人有没有回复**？如果那一轮机器人没有产生回复（比如被冷却跳过了），就不会追加 @。
4. **是不是在冷却中**？刚触发过 @ 的同一句话，在冷却时间内不会再 @。
5. **问句太短**？默认 6 个字以下的话不算。
6. **不在值班时间**？如果设了排班，当前不在班就不会 @。
7. **看日志**：打开 `logs/app.log`，搜索「人工转接」，能看到详细的判断过程。

### 转发了但客服没收到？

- 客服必须**先和机器人账号有过私聊**（至少发过一条消息或点过 Start），否则 Telegram 不允许发私聊。
- 如果客服拉黑了机器人账号，也收不到。
- 日志里搜「转发至客服」看具体报错。

### 怎么测试？

1. 在群里连续发同一句话（比如「你好你好你好」发 3 遍）。
2. 如果机器人每次都回复了，第 3 次回复后面应该会带上 @ 客服的内容。
3. 注意：如果机器人有「相同内容冷却」的设置，可能会导致连发同一句话时机器人不回复。建议测试时把 `skills.cooldown.per_content` 设成 0。

## 六、设置页速览

打开后台 → 人工客服转接：

- **精简视图**（默认）：只显示最常用的设置，三步搞定。
- **完整视图**：显示全部设置，包括多客服表格、分组排班、JSON 编辑等。

| 区域 | 内容 |
|------|------|
| ① 开关与触发 | 启用开关、触发次数、时间窗口、冷却 |
| ② 人工客服 | 客服用户名、user_id |
| 值班/排班（折叠） | 值班模式、时区、快速场景按钮 |
| 话术（折叠） | 转发开关、定位按钮开关、非值班提示、@ 前说明文案 |
| ③ 多名客服（完整视图） | 客服表格 + JSON 编辑 |
| ④ 分组（完整视图） | 分组排班 JSON |
| ⑤ 分队策略（完整视图） | 多组命中时怎么选、轮询范围 |
| ⑥ 周模板（完整视图） | 每周上班时间 JSON |
| ⑦ 按日例外（完整视图） | 节假日等特殊日期 |
| ⑧ @ 与展示（完整视图） | 多客服之间的分隔符、@ 方式 |
| 验证与排班自检 | 保存后点这里确认配置已生效 |
| 手动值班 | 手动开关当前是否值班 |

## 七、配置文件参考

所有设置都保存在 `config/config.yaml` 的 `human_escalation` 部分。Web 后台修改后会自动写入这个文件。

完整配置项和示例见 `config/config.example.yaml`。
