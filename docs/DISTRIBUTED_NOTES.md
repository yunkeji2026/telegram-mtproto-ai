# 多实例与部署形态说明

## 单进程假设

本项目的持久化以 **本机 SQLite**（配置目录下 `*.db`）为主，包括审计、知识库、情景记忆向量库等。**同一套 `config/` 目录不应被多个进程同时写入**；多实例并行跑同一数据目录会导致锁竞争、损坏风险或未定义行为。

## 若需水平扩展

- **读写分离**：仅只读副本可共享同一静态知识库文件（仍需避免与写入进程并发写同一库）；更稳妥是为只读侧做定期导出或复制到只读介质。
- **多机**：每台机器独立 `config/` 与数据库；用上游负载均衡分发 Telegram/Web 流量时，注意 **会话与状态不跨节点自动同步**（除非另行引入共享存储或同步服务）。
- **Web 管理端**：多 worker（如 gunicorn 多进程）若共用一个 `audit.db`，SQLite 在并发写入下可能成为瓶颈；单机多 worker 一般可用，高并发写入建议减少审计写入频率或换存储方案（属自定义扩展）。
