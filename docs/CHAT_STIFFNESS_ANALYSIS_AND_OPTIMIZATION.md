# 聊天呆板原因分析与优化方案

## 一、从配置与代码推断的呆板原因

### 1. 情绪增强器被关闭（影响最大）

- **现状**：`config/config.yaml` 中 `emoticons.enabled: false`（注释为「临时禁用情绪增强功能（修复空格问题）」）。
- **影响**：
  - 没有**语气微调**：如「您好」→「您好呀」、「好的」→「好的呢」、「可以」→「可以的呀」等不会生效。
  - 没有**适度表情**：不会在合适场景加 😊、👍、📦 等，整体偏「公文腔」。
  - 消极情绪时不会自动加「非常抱歉给您带来不便」等缓冲句。
- **结论**：情绪增强关闭后，回复缺少人情味和节奏变化，是呆板感的主要来源之一。

### 2. 系统提示词偏「条款化」

- **现状**：`ai.system_prompt` 大量使用「必须遵守」「禁止」「回复原则」等硬性条款，且订单类话术给的是固定句式示例。
- **影响**：模型容易倾向于「安全、模板化」输出，不敢用口语化、轻微变化，导致多轮对话听起来像复制粘贴。
- **结论**：在保证业务正确的前提下，需要明确允许「语气自然、有轻微变化」。

### 3. 「回复简洁」被过度执行

- **现状**：系统提示里写「回复简洁，避免长篇解释」。
- **影响**：容易理解为「越短越好」，出现只回一句短句、缺少衔接语（如「嗯」「好的」「那这边……」），显得生硬。
- **结论**：应改为「简洁但完整、带一句衔接」，避免单句甩结论。

### 4. 意图与技能回退到固定模板

- **现状**：各 Skill 在 AI 失败或无回复时会回退到 `_get_template_reply`（如 greeting、order_query、small_talk），从配置里随机抽一句模板。
- **影响**：模板数量有限且风格统一，多次触发同一种意图时容易重复同一句或几句，显得机械。
- **结论**：可增加模板数量、或对「无 AI 回复」时再做一次轻量生成而不是纯模板。

### 5. 温度与生成长度

- **现状**：`temperature: 0.75`，`max_tokens: 768`。
- **影响**：温度不算低，若上面几条不改，仅调高温度对「呆板」改善有限；max_tokens 足够，不是主要瓶颈。
- **结论**：可微调到 0.8 增加变化，但优先改提示词与情绪增强。

### 6. 情绪增强器本身的「空格问题」

- **现状**：情绪增强器里的 `_cleanup_format` 使用 `[\u263a-\U0001f9ff]` 等正则，在部分环境下可能误伤中文等字符，导致出现异常空格而被整体关闭。
- **结论**：若重新开启情绪增强，需要收紧或替换 emoji 匹配规则，避免误伤正常文字。

## 二、优化方案（按优先级）

### 方案 A：在不动情绪增强器的前提下，先改系统提示词（立刻可做）

**目标**：让模型主动「语气自然、有轻微变化」，又不影响订单/查单等 SOP。

在 `config/config.yaml` 的 `ai.system_prompt` 中，在「人设与风格」或「回复原则」附近增加一条：

```yaml
# 在「人设与风格」下增加一条，例如：
- 回复要有轻微变化，避免每句都像复制粘贴；可适当用口语化表达如「呢」「啦」「哈」「好的呀」，但保持专业。
```

并将「回复简洁」一句改为更明确的表述，例如：

```yaml
# 原：回复简洁，避免长篇解释
# 改为：
2. 回复简洁但完整：一句话能说清就说清，但带上自然衔接（如「好的」「那这边……」），避免只甩一句结论显得生硬。
```

**预期**：在不重新开启情绪增强的前提下，先减轻「每句都像模板」的感觉。

---

### 方案 B：修复情绪增强器并重新开启（推荐）

**目标**：恢复语气微调与适度表情，同时避免空格问题。

1. **定位并收窄 emoji 正则**  
   在 `src/skills/emotion_enhancer.py` 的 `_cleanup_format` 中，将「在文字与表情之间加空格」的正则改为更保守的实现，例如：
   - 只对「已知常用 emoji 字符」做前后空格处理（例如用明确的一小段 Unicode 范围或字符列表），避免用超大范围 `\u263a-\U0001f9ff` 误伤中文。
   - 或暂时去掉「表情与文字之间加空格」的步骤，只保留「多余空白合并」：`reply = re.sub(r'\s+', ' ', reply.strip())`。

2. **重新开启配置**  
   在 `config/config.yaml` 中设 `emoticons.enabled: true`，保存后重启。

3. **可选**：在 `emotion_enhancer` 中为「添加表情」增加开关或概率，例如仅 70% 概率添加表情，避免每条都带表情反而显得刻意。

**预期**：语气更自然、适度表情回归，明显减轻呆板感。

---

### 方案 C：提高回复多样性（温度与意图层）

1. **温度**：将 `ai.temperature` 从 `0.75` 提到 `0.8`（若希望更稳可保持 0.75，先观察 A/B 效果）。
2. **模板回退**：在 `config/templates`（或当前模板配置）中为 greeting、order_query、small_talk 等增加 2～3 种同义不同表述的模板，减少同一意图下重复同一句的概率。
3. **意图 small_talk**：若当前短句多数落在 small_talk，可考虑对「无 AI 回复」的 small_talk 再做一次「只生成一句简短回复」的轻量调用，而不是纯随机模板。

**预期**：同一类问题多次出现时，回复不再那么「总是一句话」。

---

### 方案 D：可观测与迭代

1. **日志**：对「本次回复是否来自模板」打标记（例如在 skill_manager 或 telegram_client 里打 `reply_source: template | ai`），便于后续看模板占比。
2. **抽样**：定期看几条「用户消息 + 实际回复」，重点看是否仍有「您好」「好的」「已收到」等千篇一律开头，再决定是否继续加强提示词或情绪规则。

---

## 三、建议实施顺序

1. **先做方案 A**：改系统提示词（增加「语气自然、有轻微变化」+ 修正「回复简洁」表述），不依赖情绪增强，风险小。
2. **再做方案 B**：修情绪增强器中的空格问题并重新开启，预期收益最大。
3. **视效果做方案 C**：调温度、补模板、或 small_talk 轻量生成。
4. **方案 D**：作为长期习惯，便于持续优化「呆板」感。

---

## 四、涉及文件一览

| 项目         | 文件/配置 |
|--------------|-----------|
| 情绪增强开关 | `config/config.yaml` → `emoticons.enabled` |
| 系统提示词   | `config/config.yaml` → `ai.system_prompt` |
| 温度/长度    | `config/config.yaml` → `ai.temperature`, `ai.max_tokens` |
| 情绪增强逻辑 | `src/skills/emotion_enhancer.py`（`_cleanup_format`、`_adjust_tone`、`_add_emoticons`） |
| 模板回退     | `src/skills/skill_manager.py`（`_get_template_reply`）及模板配置 |

上述分析与方案可直接用于迭代配置和代码；若你提供一段「用户消息 + 实际回复」的日志样例，可以再针对单条对话做更细的呆板原因与改写建议。
