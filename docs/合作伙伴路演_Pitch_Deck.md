# 智能客服助手 · 合作伙伴路演稿（文字版）

> 面向非技术背景的合作伙伴：全文无代码、无系统名词，用大白话说明价值与合作方式。

---

## 1️⃣ 一句话概括

**这是一套能代替人工在聊天里回答客户问题的「智能客服助手」**：客户问订单、问规则、问进度时，它能马上给出统一、准确、礼貌的回复，**把重复劳动从人身上拿掉**，让团队把时间花在真正要紧的事上。

---

## 2️⃣ 痛点与场景故事

**小王是一家跨境业务团队里的一线同事。**  
每天群里消息不断：有人发订单号问进度，有人问「这个通道今天额度多少」，有人半夜来催单。小王要一边翻记录、一边查表格、一边担心说错话，**吃饭和睡觉都被消息打断**。主管还要求口径统一，可新人一上手就容易答错，客诉和内部扯皮都多了。

**用上这套助手之后：**  
常见问题的答案由助手先接住，**口径和数字跟后台设定走**，小王只在复杂或特殊情况时出面。客户感觉**回复更快、更稳**，小王终于能按计划休息，团队也不用再为「谁又说错了」吵架。

**合作伙伴一听就能懂：** 这不是换一套软件那么简单，而是**把「人海战术」换成「人机配合」**，省人、省心、少纠纷。

## 3️⃣ 产品功能说明

| 核心能力 | 通俗比喻 | 对伙伴的价值 |
|----------|----------|----------------|
| **自动应答** | 像一位 **24 小时在线、从不请假** 的一线同事，按你们设定好的话术和规则回答。 | 减少夜班与重复排班，人力成本可控。 |
| **订单与业务问询** | 像前台 **先问清要什么、再引导补材料**，该确认的确认，不该瞎编的一句不乱说。 | 降低错答、漏答带来的客诉与赔付风险。 |
| **规则与「额度/通道」类说明** | 像一块 **随时更新的告示牌**：你们改数字或状态，对外说法跟着变，不用每人背新版本。 | **日常维护不需要程序员**，运营或主管改设置即可，**后期不绑死技术团队**。 |
| **知识沉淀与学习** | 像一本 **越用越厚的答复手册**：新问题可以整理成标准答案，团队共用。 | 新人上手快，老人少重复，**降低培训与交接成本**。 |
| **管理后台** | 像给主管用的 **控制台**：看整体情况、调展示内容，不必懂开发。 | 合作方或客户方可参与运营，**分工清晰**。 |

**强调一点：** 日常改规则、改话术、改展示，**按界面操作即可**，不需要长期养专职技术员。

## 4️⃣ 市场与竞品分析

**目标用户是谁？**  
- 在**即时聊天**里做客服、做商户支持、做订单与资金类咨询的团队（含外包客服公司、支付/跨境/贸易类企业、需要多群运营的品牌方等）。  
- **市场规模**：全球企业客服与联络中心是千亿人民币量级的年市场；其中**大量重复咨询**适合用「人机协作」先筛一层——哪怕只切一小块垂直场景，空间也足够做成一门可持续的生意。

**简单对比（让差异一眼看懂）：**

| 常见做法 | 我们的优势 |
|----------|------------|
| **纯人工坐席**：全靠人回消息，成本高、波动大、夜班难招。 | 重复问题先由助手接住，**同样人数能服务更多客户**，情绪与口径更稳。 |
| **传统外包客服**：按人头或按席计费，单价清晰但**难随业务量弹性伸缩**。 | 业务量上去时**主要靠规则与内容扩容**，不必线性加人；**维护以运营为主**，不必堆研发。 |
| **只做简单机器人的方案**：只能答固定几句，一问深就崩，客户体验差。 | 在「固定规则 + 可更新的业务说明」之间做好平衡，**该严的地方严、该活的地方活**，减少「答非所问」。 |

**一句话总结给伙伴：** 我们追求的是 **更省人、更可控、上线后你们自己能玩得转**，而不是卖一套复杂系统让你们长期依赖原厂工程师。

## 5️⃣ 成功里程碑（关键指标）

上线后，我们会一起盯这些数，**让成功可被衡量**：

| 指标类型 | 说明（示例方向） |
|----------|------------------|
| **采用与活跃** | 多少群/多少客户场景接入了助手；**日均有效对话量**是否稳步上升。 |
| **人工减负** | 一线同事**人均处理消息量**提升，或同样业务量下**所需人手下降**（二者至少满足其一）。 |
| **满意度与投诉** | 与「答错、答慢」相关的投诉**环比下降**；抽样满意度或好评率提升。 |
| **留存与续约** | 使用方**持续使用意愿**（续约、扩场景）；合作伙伴侧**复购或增购**。 |
| **商业结果（与伙伴相关）** | 按合作模式约定：**分润比例、分成结算周期、首年目标签约客户数或流水**等，在签约时写清里程碑。 |

*具体数字（例如首年目标用户数、分润点位）建议在正式商务谈判中按区域与行业单独约定。*

## 6️⃣ 商业模式

盈利可以多种方式组合，典型包括：

1. **订阅制**：按年/按月收取「使用费」，包含一定对话量或场景数，超出部分加价。  
2. **按效果或按量**：按有效会话量、按接入账号或按席位阶梯计费。  
3. **交易分成**：与支付、撮合、流水相关的场景，可按约定比例从**增量业务**中分成。  
4. **增值服务**：培训、驻场运营、行业模板包、专属话术打磨等。

*实际以双方签署的协议为准；可先小范围试点再放大。*

## 7️⃣ 合作模式（具体选项）

| 选项 | 适合谁 | 您大概怎么赚钱 / 怎么合作 |
|------|--------|---------------------------|
| **A. 渠道代理模式（按销售分成）** | 有客户资源、懂行业痛点的伙伴 | 您负责触达与签约，我们负责交付与续费支持；**按合同约定的销售分成或返点**结算。 |
| **B. 按使用量付费（调用型）** | 已有平台、希望把能力嵌进自家产品 | 按**对话量、账号数或调用档位**计费；您向终端客户打包报价，**差额即您的空间**。 |
| **C. 定制开发 / 联合运营** | 单一大客户或区域独家场景 | **一次性项目费**或**联合运营、收入分成**；深度贴合对方流程，可谈**合资或排他期**。 |

三种方式可并行探讨：例如先做 **A** 打开市场，成熟客户转 **C** 做深。

## 8️⃣ 下一步行动

**短期（1～3 个月）**  
- 选定 **1～2 个试点客户或场景**，明确成功标准（减负比例、满意度、是否续约）。  
- 敲定 **合作模式与分润/计费** 原则，签署框架协议。  
- 安排 **对接人**：我方负责产品与交付节奏，贵方负责客户侧协调与验收。  

**中期（约半年）**  
- 试点复盘，形成 **可复制的行业话术与包装**，便于您批量拓展。  
- 视情况扩展 **区域或行业代理**，约定 **目标与激励**。  
- 探索 **联合品牌或联合方案**，在展会、渠道会上一起露面。  

**需要合作伙伴配合的事**  
- 提供 **真实客户场景与反馈**，便于我们把产品磨到「好用、敢承诺」。  
- 在商务与回款上 **按约定推进**，保证试点可持续。  
- 若走代理模式，请指定 **日常联络窗口**，便于联合拜访与售后。
