# 测试功能与流程说明

> 这份文档告诉你怎么测试每个功能是否正常工作，不需要懂代码，按步骤操作即可。

---

## 测试前准备

**需要的东西：**
1. 已启动的机器人（服务器上运行 `python main.py`）
2. 能打开的后台网页（浏览器输入地址，如 `http://服务器IP:8080`）
3. 一个 Telegram 测试群（机器人已在群内）
4. 后台登录账号（用户名 + 密码）

**确认机器人正在运行：**
- 打开后台 → 左下角显示"运行中"（绿色）
- 或打开"实时日志"页面，看到有日志在滚动

---

## 模块一：后台登录

### 测试步骤
1. 打开后台网址
2. 输入**正确的用户名和密码**，点登录
3. 应该跳转到仪表盘首页 ✅
4. 重新打开登录页，输入**错误密码**，点登录
5. 应该显示"密码错误"或"账号不存在" ✅
6. 不登录直接访问 `http://服务器IP:8080/templates`
7. 应该被自动跳回登录页 ✅

**测试通过标准：** 正确账号能进、错误账号进不去、未登录无法访问内页。

---

## 模块二：仪表盘

### 测试步骤
1. 登录后查看仪表盘
2. 确认顶部数字卡片有数据（运行时长 > 0）✅
3. "今日活动"图表能正常显示（柱状图 + 圆形图）✅
4. 点击"系统诊断"按钮
5. 弹出检查结果弹窗，显示评分（0-100）和问题列表 ✅
6. 关闭弹窗后再次点击"系统诊断"，数据刷新 ✅

**测试通过标准：** 数据正常显示，诊断弹窗能打开/关闭。

---

## 模块三：话术模板

### 测试步骤 — 查看和修改

1. 点左侧菜单"话术模板"
2. 看到话术列表，每条有键名和内容 ✅
3. 随便点一条话术的内容文本框，修改一些文字
4. 点"保存"按钮
5. 应该弹出**变更预览弹窗**，左边显示原来的内容，右边显示新内容 ✅
6. 点"确认保存"
7. 页面出现绿色"保存成功"提示 ✅
8. 刷新页面，刚才改的内容仍然保留 ✅

### 测试步骤 — 批量删除

1. 勾选 2-3 条话术左边的复选框
2. 底部出现"批量删除"悬浮栏 ✅
3. 点"批量删除"
4. 弹出确认对话框（询问是否确认） ✅
5. 确认后，被选中的话术消失 ✅

**测试通过标准：** 修改能保存、有变更预览、批量删除有确认步骤。

## 模块四：通道管理

### 测试步骤 — 修改通道状态

1. 点左侧菜单"通道管理"
2. 找到一个通道，将状态改为"维护中"
3. 点"保存"按钮
4. 弹出变更预览（显示状态从"正常"变为"维护中"）✅
5. 确认保存后，刷新页面，状态已更新 ✅

### 测试步骤 — 批量操作

1. 勾选多个通道左边的复选框
2. 底部出现批量操作栏 ✅
3. 点"批量维护"
4. 弹出确认弹窗 ✅
5. 确认后，所有选中通道状态变为"维护中" ✅

**测试通过标准：** 单条修改和批量修改都能正常保存。

## 模块五：策略管理

### 测试步骤 — 启用/禁用

1. 点左侧菜单"策略管理"
2. 找到一个策略卡片，点击"启用/禁用"开关
3. 开关颜色变化 ✅
4. 点"保存"按钮，显示保存成功提示 ✅

### 测试步骤 — 查看变更历史

1. 点任意策略卡片右下角的"历史"按钮
2. 弹出时间轴弹窗 ✅
3. 如果该策略有改过，会显示几条记录（时间、操作人、操作内容）✅
4. 如果从未修改过，显示"暂无变更记录" ✅
5. 点右上角"×"关闭弹窗 ✅

### 测试步骤 — 批量操作

1. 勾选多个策略卡片左上角的复选框
2. 底部浮现"批量启用" / "批量禁用" / "取消" 按钮 ✅
3. 点"批量禁用"
4. 弹出确认弹窗 ✅
5. 确认后，所有选中策略状态改变 ✅

**测试通过标准：** 开关能切换、历史弹窗能打开、批量操作有确认步骤。

## 模块六：操作记录（审计日志）

### 测试步骤

1. 先去"话术模板"随便改一条内容并保存
2. 再点左侧菜单"操作记录"
3. 应该能看到刚才的操作记录（操作人 = 你的账号名）✅
4. 用顶部筛选器：选择"操作类型 = 更新"或输入你的账号名
5. 点"查询"，列表应该只显示匹配的记录 ✅
6. 点"导出"按钮，下载 CSV 文件
7. 用 Excel 打开，内容不乱码，第一行是导出时间和筛选条件 ✅

**测试通过标准：** 操作可追溯、筛选正常、导出 CSV 能用 Excel 打开。

---

## 模块七：版本对比与回滚

### 测试步骤 — 版本对比

1. 先去"话术模板"改几条内容并保存（产生快照）
2. 点左侧菜单"版本对比"
3. 左侧时间轴应该显示刚才的快照 ✅
4. 点击一个快照，它被选中为"版本A" ✅
5. 再点另一个快照，选中为"版本B" ✅
6. 点"比较"按钮，右侧显示差异（绿=新增，红=删除）✅

### 测试步骤 — 回滚

1. 在版本对比页面选中一个历史快照
2. 点"回滚"按钮
3. 弹出确认弹窗（警告此操作会覆盖当前配置）✅
4. 确认后，显示回滚成功提示 ✅
5. 去"话术模板"页面验证内容已恢复 ✅

**⚠️ 注意：** 回滚是真实生效的，测试时建议先导出当前配置备份。

**测试通过标准：** 时间轴显示快照、对比能看到差异、回滚后配置实际恢复。

## 模块八：导入 / 导出

### 测试步骤 — 导出

1. 打开"导入/导出"页面（从仪表盘快速操作按钮或直接导航）
2. 点"ZIP 打包"按钮
3. 浏览器自动下载一个 zip 文件 ✅
4. 解压 zip，里面有 yaml 配置文件 ✅

### 测试步骤 — 导入（覆盖模式）

1. 在"导入"区域点"覆盖模式"选项
2. 把刚才下载的 zip 文件拖入上传框（或点击选择文件）
3. 文件名和大小显示在预览区域 ✅
4. 按钮显示"覆盖导入" ✅
5. 点按钮，进度条走完 ✅
6. 页面显示导入成功 ✅

### 测试步骤 — 导入（增量合并模式）

1. 选择"增量合并"选项
2. 上传同一个 zip 文件
3. 按钮变为"增量合并导入" ✅
4. 导入成功后，提示里显示"+N 条新增 / ~M 条更新" ✅

### 测试步骤 — Webhook 通知

1. 在页面下方找到"Webhook 变更通知"卡片
2. 填入一个可接收 POST 请求的网址（可用 https://webhook.site 测试）
3. 勾选"启用 Webhook"，点"保存设置" ✅
4. 点"发送测试"按钮
5. 在 webhook.site 上能看到收到的 JSON 请求 ✅

**测试通过标准：** 导出有文件、导入成功、增量模式有统计、测试通知能收到。

## 模块十一：Bot 自动回复（Telegram 群内测试）

> 需要在有机器人的 Telegram 测试群里操作。

### 测试步骤 — 关键词触发

1. 在群里发消息：`订单还没到`
2. 机器人应该在几秒内自动回复 ✅
3. 发消息：`check status`（英文）
4. 机器人也应该回复 ✅
5. 发一段不相关的内容（如"今天天气不错"）
6. 机器人**不应该回复**（不是客服相关话题）✅

### 测试步骤 — @提及触发

1. 在群里发：`@机器人用户名 我的订单怎么了`
2. 机器人应该立即回复 ✅

### 测试步骤 — 图片识别（订单号）

1. 在群里发一张包含**订单号或交易流水号的截图**（如付款凭证）
2. 机器人应该回复并提到图片里识别出的单号或金额 ✅
3. 发一张不含订单信息的普通图片
4. 机器人回复时**不应该编造不存在的单号** ✅

### 测试步骤 — 连续追问

1. 先发一条触发机器人回复的消息（如"查一下我的订单"）
2. 机器人回复后，再发一条追问：`还没处理吗`
3. 机器人应该能理解这是追问，并继续回复（而不是重新要求提供订单号）✅

**测试通过标准：** 关键词/提及能触发、不相关内容不乱回、图片能识别单号、追问有上下文记忆。

## 快速故障排查

| 现象 | 先检查什么 |
|------|-----------|
| 机器人不回复 | 后台"实时日志"是否有错误日志；配置里 `process_groups` 是否为 true |
| 图片识别失败 | "实时日志"搜索"Vision"或"OCR"看有无错误；确认 API key 是否有效 |
| 后台打不开 | 确认机器人服务正在运行；端口是否被防火墙拦截 |
| 回滚后配置没变 | 查"操作记录"确认回滚操作是否成功；重启机器人使配置生效 |
| 导入后没效果 | 机器人需要重启才能加载新配置 |
| 某账号登录失败 | "用户管理"里确认账号是否被禁用 |
