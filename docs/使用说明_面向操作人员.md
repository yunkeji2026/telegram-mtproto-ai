# 智能客服系统（Camille）— 使用说明（面向操作人员）

本文档面向**日常使用与配置维护人员**，说明如何运行系统、如何修改常用配置、如何通过对话或文件改额度/黑名单，以及常见问题处理。更完整的功能列表见《功能总结与使用说明》。

---

## 一、运行与停止

### 1.1 启动前准备

- 已安装 Python 3.8 及以上。
- 已在项目目录执行过 `pip install -r requirements.txt`。
- 已按《功能总结与使用说明》配置好 `config/config.yaml`（Telegram 账号、AI API 密钥等）。

### 1.2 启动程序

在项目根目录执行：

```text
python main.py
```

首次运行如提示手机验证或两步验证，按终端提示在手机 Telegram 中完成即可。会话会保存在项目目录，下次启动无需再验证。

### 1.3 停止程序

在运行 `main.py` 的终端按 `Ctrl+C` 正常退出。

### 1.4 查看是否正常

- 终端有「AI聊天助手初始化完成」「Telegram客户端初始化成功」等提示即表示启动成功。
- 若开启了监控，浏览器访问 `http://127.0.0.1:9090/api/health`（端口以配置为准）可查看运行状态。

## 二、机器人会做什么、不会做什么

- **会做的**：在私聊和已加入的群聊中，根据关键词、@、回复关系等判断是否回复；回复内容包含订单查询引导、通道/额度说明（按配置的 EP/JC 数字与特殊群/黑名单）、多语言话术与情绪语气调整等。
- **不会做的**：不会主动给陌生人发消息；不会在未触发的群消息下回复；不会编造订单金额/单号（无凭证时只引导发订单号或截图）。

触发条件（满足其一即可回复）：被 @、命中配置的关键词（如订单、额度、客服等）、回复了机器人自己的消息、或经过多轮判断认为需要回复。详见配置中的 `trigger` 与 `group_reply`。

## 三、常用配置修改（不改代码）

以下均为**改配置文件即可**，保存后部分项热更新（额度规则、部分主配置），部分需重启 `main.py` 后生效。

### 3.1 额度数字与群名单（热更新，无需重启）

- **文件**：`config/quota_rules.yaml`
- **可改项**：
  - **通道额度**：在 `channels` 下改 `ep` / `jc` 的 `default_range`、`special_range`（如 100-20000）。
  - **特殊客户群**：在 `special_groups` 下列出群名，**必须与 Telegram 群标题完全一致**（含空格、大小写）。
  - **黑名单群**：在 `blacklist_groups` 下按格式写「群名 → ep/jc 话术」；留空 `{}` 表示无黑名单。
  - **额度说明补充**：在 `quota_note_supplement` 下写一段话（如用尽恢复、每日限额变化），会追加在额度回复末尾。
  - **引导语**：用户未说查哪个通道时，可改 `templates.ask_channel` 的文案（建议保留「额度/限额」表述）。

保存后，下次有人问额度时会自动用新配置，**无需重启**。

### 3.2 通过和机器人对话改额度配置（需先开启）

- **前提**：在 `config/config.yaml` 的 `telegram.quota_config_commands` 中：
  - `enabled` 设为 `true`
  - `allowed_user_ids` 中填入允许操作的 Telegram 用户 ID（可填多个）
- **用法**：用该账号在私聊或群聊中对机器人发送**正常一句话**即可，例如：
  - `列出特殊群` / `列出黑名单`
  - `添加特殊群 群名` / `删除特殊群 群名`
  - `添加黑名单 群名` / `删除黑名单 群名`
- 执行后会直接改 `quota_rules.yaml` 并热更新；群名需与 Telegram 群标题完全一致。

未配置或非允许用户会收到「无权限」或不会执行配置修改。

### 3.3 群内何时回复、不回复谁

- **文件**：`config/config.yaml`
- **群回复模式**：`telegram.group_reply.mode` 可设为 `mention_or_keyword`（被 @ 或命中关键词才回）、`mention_only`、`keyword_only`、`always` 等。
- **不回复的账号**：在 `telegram.no_reply_sender_usernames` 下列出用户名（如 `gxp_notify_bot`），这些账号发的消息不会触发回复。
- **被 @ 名单**：`telegram.group_reply.mention_usernames` 中填写会被视为「在叫我们」的 @ 名或显示名。

修改后一般需**重启** `main.py` 生效（具体以各配置项说明为准）。

### 3.4 日志详细程度

- **文件**：`config/config.yaml` → `logging.level`
- 设为 `DEBUG` 可看到额度分支的群名匹配情况（chat_title、special、blacklist），便于核对配置是否生效；日常可用 `INFO`。

## 四、常见问题

| 现象 | 可能原因 | 建议操作 |
|------|----------|----------|
| 群里不回复 | 未 @、未命中关键词、或触发逻辑未通过 | 检查 `group_reply.mode` 与 `trigger_rules.yaml` 关键词；将 `logging.level` 设为 `DEBUG` 看是否触发 |
| 额度数字不对 | 配置未生效或改错文件 | 确认改的是 `config/quota_rules.yaml`，保存后无需重启；查日志是否有「额度规则已加载」 |
| 特殊群/黑名单没生效 | 群名与 Telegram 群标题不一致 | 把群标题**完整复制**（含空格、符号）到 `special_groups` 或 `blacklist_groups`；DEBUG 日志可看当前匹配到的 chat_title |
| 对话命令改配置没反应 | 未开启或用户不在白名单 | 检查 `quota_config_commands.enabled` 为 `true` 且自己的 Telegram 用户 ID 在 `allowed_user_ids` 中 |
| 程序启动报错 | 缺依赖或配置错误 | 执行 `pip install -r requirements.txt`；检查 `config.yaml` 中 api_id、api_hash、api_key 等是否填写正确 |

## 五、管理面板 — 简洁模式 / 完整模式

后台管理面板支持两种界面模式，方便不同角色的用户使用。

### 5.1 切换方式

- **顶栏模式徽章**：点击右上角的「简洁」/「完整」徽章即可切换。
- **命令面板**：按 `Ctrl+K` 打开命令面板，输入 `#simple` 或 `#full` 快速切换。
- **URL 参数**：访问 `/set_ui_mode?mode=simple` 或 `/set_ui_mode?mode=full` 直接切换。

### 5.2 简洁模式（推荐客服人员使用）

简洁模式隐藏大量高级功能，只保留日常工作必需的页面和操作：

| 页面 | 简洁模式下的变化 |
|------|-----------------|
| **待处理案例** | 隐藏链路分析、满意度条；只显示核心操作（备注、结案） |
| **知识库** | 隐藏高级统计、批量操作、翻译、沙盒；保留查看和基础编辑 |
| **AI 学习** | 隐藏已通过/已拒绝统计；聚焦待审核草稿 |
| **常用指令** | 隐藏管理员/超管命令，只显示客服常用指令 |
| **数据概览** | 隐藏系统指标、Bot 指标等技术图表；显示快捷入口卡片 |
| **数据分析** | 隐藏刷新条、周期选择器、部分图表 |
| **操作日志** | 隐藏活动热力图和导出按钮 |
| **通道管理** | 只读模式，显示通道状态摘要（总数/在线/禁用），隐藏编辑按钮 |
| **系统设置** | 只显示「AI 提示词」和「人工转接」两个区段，隐藏 API、安全、Bot 行为等 |
| **AI 策略** | 只读概览，隐藏参数滑块、保存按钮、批量操作 |
| **话术模板** | 只读模式，可查看模板内容但隐藏编辑和批量操作 |

侧边栏也会简化：核心页面（案例、知识库、学习、指令）直接显示，其余折叠到「更多功能」。

### 5.3 完整模式（管理员/开发者使用）

完整模式显示所有功能，包括：
- 所有配置页面（系统设置、AI 策略、策略效果、用户管理、导入导出、版本对比、实时日志）
- 批量操作（全选、批量删除、批量启用/禁用）
- 高级统计和图表
- JSON 编辑器和调试工具

### 5.4 默认模式

| 角色 | 默认模式 |
|------|---------|
| 主帐号（master） | 完整模式 |
| 管理员（admin） | 完整模式 |
| 只读观察员（viewer） | 简洁模式 |

用户可随时手动切换，偏好保存在浏览器 Cookie 中。
