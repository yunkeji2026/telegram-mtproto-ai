# 部署到新服务器指南

## 一、需要准备的东西

### 1. 新服务器要求
- Python 3.11 或以上
- 系统：Windows / Linux / macOS 都可以
- 内存：至少 2GB（推荐 4GB）
- 硬盘：至少 1GB 可用空间
- 网络：能访问 Telegram 和 AI API

### 2. 新系统需要的账号信息
- **Telegram API**：到 https://my.telegram.org 申请新的 `api_id` 和 `api_hash`（每个手机号一套）
- **AI API Key**：Gemini / DeepSeek / OpenAI 的 API 密钥
- **Telegram 手机号**：新系统要用的机器人手机号（不能和现有的相同，否则会冲突）
- （可选）智谱 GLM API Key：图片理解功能用的

## 二、复制哪些文件

### 必须复制的（核心代码）
```
telegram-mtproto-ai/
├── main.py                    # 主程序入口
├── requirements.txt           # Python 依赖
├── src/                       # 全部源代码（整个目录）
│   ├── ai/
│   ├── client/
│   ├── context/
│   ├── monitoring/
│   ├── skills/
│   ├── trigger/
│   ├── utils/
│   ├── web/
│   ├── image_recognizer.py
│   ├── vision_client.py
│   └── voice_transcriber.py
├── domains/                   # 域包（整个目录）
│   ├── payment/               # 支付行业域包
│   └── ecommerce/             # 电商行业域包（示例）
├── plugins/                   # 插件目录
└── config/
    └── config.example.yaml    # 配置模板
```

### 不要复制的（每台机器独有）
```
❌ config/config.yaml          # 包含密钥，新服务器要重新配
❌ config/*.db                 # 数据库文件，新系统会自动创建
❌ sessions/                   # Telegram 登录会话，新号要重新登录
❌ logs/                       # 日志文件
❌ __pycache__/                # Python 缓存
❌ .pytest_cache/              # 测试缓存
❌ temp/                       # 临时文件
```

### 可选复制的（看需求）
```
⚡ config/exchange_rates.yaml   # 汇率配置（如果新系统也是支付行业）
⚡ config/quota_rules.yaml      # 额度规则（同上）
⚡ config/trigger_rules.yaml    # 触发规则（可能要调整）
⚡ config/templates.yaml        # 回复模板
⚡ config/emoticons.yaml        # 表情包配置
⚡ config/knowledge_base.db     # 知识库（如果想带过去旧的知识条目）
```

## 三、新服务器操作步骤

### 第 1 步：安装 Python
```bash
# Linux (Ubuntu/Debian)
sudo apt update && sudo apt install python3.11 python3-pip

# Windows
# 下载安装 https://www.python.org/downloads/
```

### 第 2 步：上传代码
把上面列的"必须复制的"文件打包上传到新服务器：
```bash
# 在当前服务器打包（排除不需要的文件）
cd telegram-mtproto-ai
tar --exclude='config/config.yaml' \
    --exclude='config/*.db*' \
    --exclude='sessions/*.session*' \
    --exclude='logs/*.log' \
    --exclude='__pycache__' \
    --exclude='.pytest_cache' \
    --exclude='temp/*' \
    -czf ../deploy-package.tar.gz .

# 上传到新服务器
scp ../deploy-package.tar.gz user@新服务器IP:/opt/telegram-ai/
```

Windows 可以直接用压缩软件打包，排除上述目录后上传。

### 第 3 步：安装依赖
```bash
cd /opt/telegram-ai
pip install -r requirements.txt
```

### 第 4 步：创建配置文件
```bash
cp config/config.example.yaml config/config.yaml
```

然后编辑 `config/config.yaml`，**必须修改**以下内容：

```yaml
# 1. 选择行业域包
domain: payment          # 或 ecommerce，或你自定义的域名

# 2. Telegram 账号（新的！）
telegram:
  api_id: "新的API_ID"
  api_hash: "新的API_HASH"
  phone_number: "+新手机号"
  session_name: "新的会话名"

# 3. AI 配置
ai:
  provider: "gemini"          # 或 deepseek / openai
  api_key: "新的API密钥"
  model: "gemini-2.5-flash"   # 根据实际选择

# 4. 群组监控（设置要回复的群）
group_reply:
  enabled: true
  allowed_chats: [-100新群的chat_id]

# 5. 人工客服（按新系统的客服人员配置）
human_escalation:
  enabled: true
  agents:
    - user_id: 新客服的user_id
      username: "新客服username"
      display_name: "客服小王"

# 6. Web 管理后台密码
web_admin:
  auth_token: "设一个新密码"
  secret_key: "随机生成一个密钥"
```

### 第 5 步：创建必要目录
```bash
mkdir -p logs sessions temp/images temp/voice config/snapshots
```

### 第 6 步：首次登录 Telegram
```bash
python main.py
```

首次运行会提示输入手机号验证码，完成后会在 `sessions/` 下生成会话文件。后续运行就不需要再验证了。

### 第 7 步：验证运行
```bash
# 检查是否正常启动
# 看到 "✅ Telegram客户端已启动，等待消息..." 就说明成功了

# 检查 Web 管理面板
curl http://127.0.0.1:8080/login

# 检查监控 API
curl http://127.0.0.1:9090/api/health?token=你设的密码
```

## 四、如果是不同行业

如果新系统不是支付行业，需要：

### 方案 A：用现有域包
```yaml
# config.yaml 里改域名就行
domain: ecommerce    # 切换到电商域包
```

### 方案 B：创建新域包
```
domains/
  你的行业名/
    __init__.py
    manifest.yaml        # 域包描述
    config/
      defaults.yaml      # 行业默认配置
      trigger_keywords.yaml  # 触发关键词
    kb/
      categories.yaml    # 知识库分类
      seeds.yaml         # 预设回复
    prompts/
      system_prompt.txt  # AI 系统提示词
      terminology.yaml   # 行业术语
    skills/
      __init__.py
      你的技能.py        # 行业特有技能
    i18n/
      zh.yaml            # 中文翻译
```

然后在 `config.yaml` 中设置 `domain: 你的行业名`。

## 五、常见问题

### Q: 两台服务器能用同一个 Telegram 手机号吗？
**不能。** 同一个手机号只能有一个活跃的 MTProto 会话。新服务器登录后，旧服务器会被踢下线。

### Q: 知识库要重新配吗？
如果是同行业，可以把 `config/knowledge_base.db` 复制过去，就能保留所有知识条目。如果是不同行业，建议重新通过 Web 管理面板添加。

### Q: 怎么获取群的 chat_id？
把机器人加入群后，在群里发一条消息，查看日志中的 `chat_id=-100xxxxxxxxxx`。

### Q: 端口被占用怎么办？
在 `config.yaml` 中修改：
```yaml
web_admin:
  port: 8081        # 改一个没被占的端口

monitoring:
  metrics_port: 9091  # 同上
```

### Q: 如何后台运行？
```bash
# Linux
nohup python main.py > /dev/null 2>&1 &

# 或用 systemd / supervisor / pm2 管理

# Windows
# 用 "start /b python main.py" 或注册为 Windows 服务
```
