## 2. 本产品的定位坐标

不落在上述任一格的正中央，属竞品基本未覆盖的细分：

> **面向跨境卖家「个人号矩阵」的多平台 AI 客服 + 主动获客系统，靠移动端 RPA 打通无开放 API 的渠道。**

### 2.1 独特差异化（竞品做不到/不做）

1. **个人号 RPA 是护城河**：Respond.io/Wati/SleekFlow 全部依赖**官方 WhatsApp Business API**（需企业认证、模板审核、按会话付费、**不能用个人号**）。本产品用 Android RPA 直驱**个人 LINE/WhatsApp/Messenger 账号**——这是大量东南亚/华人跨境卖家的真实作业方式（个人号加好友、私聊成交），API-only 竞品**结构性做不到**。
2. **私有化 + 数据主权**：竞品几乎全是 SaaS、不可自托管；本产品可私有部署、自带 SQLite 事实源。
3. **获客 + 客服一体**：竞品多为"客服解工单"；本产品含 contacts/journey/reactivation/intimacy 主动获客养号链路（更接近 AI-SDR + 客服合体）。
4. **零按解决量抽成**：竞品 $0.9–2/解决，规模化贵；本产品自托管 + 自带 LLM 成本追踪，边际成本仅 token。

### 2.2 诚实的差距（竞品强于我们）

| 维度 | 差距 |
|---|---|
| 解决率量化 | 竞品公布 60–90%；本产品评测框架刚建，意图规则版 **~84%**，**FAQ 解决率尚未度量** |
| 电商深度 | Gorgias「每张工单带订单上下文」成熟；本产品 Shopify connector 刚做，Woo/库存/退款判定缺 |
| 商业化 | 竞品有多租户/计量/自助开通；本产品 **E2 = 0%**，暂不可多租户售卖 |
| 合规 | 竞品走官方 API 合规；本产品 RPA 个人号有**封号/TOS 风险**（模式固有代价） |
| 成熟度 | 竞品服务数千客户；本产品单租户自用、~60% 完成度 |

## 3. 战略判断

- **不正面打 Intercom/Gorgias**：API-only、合规、SaaS 的红海，本产品无优势。
- **真实战场 = API 触达不到的个人号跨境私域**：此细分竞品几乎空白（最接近的是灰产"群控"工具，但缺 AI 客服深度/KB/风险分层/事实校验）。

### 据此排序的下一步（与 v2 §9 剩余项对齐）

1. **量化解决率**（评测框架已就位）：接 FAQ 解决率 + 实跑 LLM，用数字证明"个人号也能 AI 自动解决 X%"——对外唯一可信卖点。
2. **电商事实深度**：订单上下文注入已做（P1-b），补库存/退款判定，对标 Gorgias 的 order-context，但落在个人号渠道。
3. **若商业化**：E2 多租户是门槛，否则只能自用/单客户私有化交付。
