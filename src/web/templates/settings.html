⑤ 分队策略与排班估计

多分组命中时如何合并、以及「排班自检 / 下一开收工时间」的估算参数。与下方周模板、例外日配合使用。

使用说明
  • 分队策略：多队同时命中时，合并会去重客服；仅第一支只用最先匹配的那一队。
  • 轮询范围：在「每次只 @ 一名」时，全局共用一个计数，按会话则每个群单独轮询。
  • 粗估步长 / 细扫：数字越大越省 CPU；细扫为 0 表示只做粗网格（适合粗略展示）。
  • API 缓存：减轻 /schedule-status 压力；手动值班开关仍会即时影响是否 @。
估算「下一开/收工」时的步进；一般 15 即可。
